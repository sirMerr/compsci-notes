    Pagination¶

    Note

    Can use similar technique for "infinite scrolling" except Javascript sends Ajax query instead of a user clicking on the links

    • OFFSET and LIMIT restricts which subset of the matched records is actually returned.
    • To paginate the returned results of a query, each iteration of the page will run the same query using different OFFSET parameter.
    • The first page viewing will request the first X records; the second page viewing, the second group of X records; and so forth.
    • To make this work, two values must be used in the query
    • OFFSET, or which record to start with (where to begin fetching records)
    • LIMIT, or number of records to display -> fixed
    • The page has to know where to start -> needs to be passed to the page

    Algorithm: pagination¶

    1. decide how many records to show per page? (e.g., 10) This will be fixed

    2. Calculate / remember total number of pages based on total number of records

    3. done once, first time on page

    4. required to decide if you need prev – next links

    5. put this variable in query string so you don't have to recalculate

    6. Indicate the page you want

    7. pass this variable in query string

    8. use it to calculate the offset

    9. Total pages and current page need to be passed to script each time

    10. best way: through the GET values (query string)
